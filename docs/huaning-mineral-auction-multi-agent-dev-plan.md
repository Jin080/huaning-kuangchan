# 华宁矿产竞拍平台多 Agent 并行开发拆解方案

## 1. 目标

将华宁矿产竞拍平台拆解为多个边界清晰、可并行开发的小模块，让多个 AI Agent 同时推进开发。每个 Agent 负责独立模块，最终由集成 Agent 统一联调、验收和收口。

## 2. 总体协作方式

建议采用“先搭骨架，再并行开发，再统一集成”的方式：

1. 架构与数据模型先行：先确定项目结构、数据库表、接口规范、角色权限和状态枚举。
2. 前后端按模块并行：前台门户、企业端、后台管理、竞价核心、内容管理等模块同时开发。
3. 集成 Agent 持续合并：每天合并可运行版本，及时处理接口字段、权限、状态流转冲突。
4. 测试 Agent 全程跟进：同步编写用例，按业务流程进行端到端验证。

## 3. 模块拆解总览

| Agent | 模块 | 主要职责 | 依赖 | 预计耗时 |
|---|---|---|---|---|
| Agent 1 | 项目骨架与基础架构 | 前后端项目初始化、路由、权限框架、数据库连接、文件上传基础能力 | 无 | 2 天 |
| Agent 2 | 数据模型与状态机 | 拍品、企业、意向金、出价、成交、合同、退款、通知、日志等表结构与状态枚举 | Agent 1 | 2 天 |
| Agent 3 | 前台门户 | 首页、数据看板、拍品公告、正在竞价、成交公示、资讯入口 | Agent 1、2 | 4 天 |
| Agent 4 | 拍品与发布复核后台 | 拍品新建、编辑、草稿、发布复核、公示、关闭/取消、附件管理 | Agent 1、2 | 5 天 |
| Agent 5 | 企业入驻与认证 | 企业注册、认证资料提交、附件上传、认证状态、后台审核、驳回重提 | Agent 1、2 | 4 天 |
| Agent 6 | 意向金凭证与竞价资格 | 保证金说明、付款凭证上传、意向金复核、资格名单、驳回重传 | Agent 4、5 | 4 天 |
| Agent 7 | 竞价核心引擎 | 竞价详情、报价校验、加价幅度、最高价刷新、出价记录、截止中标确认 | Agent 2、4、6 | 6 天 |
| Agent 8 | 成交、合同、退款与黑名单 | 成交公示、合同状态、退款状态、违约标记、黑名单封禁和解除 | Agent 2、7 | 4 天 |
| Agent 9 | 个人中心与消息通知 | 我的认证、我的意向金、我的出价、站内消息、成交通知、失败通知 | Agent 5、6、7、8 | 3 天 |
| Agent 10 | 内容管理与公开说明页 | 政策法规、交易公告、矿能动态、黑名单说明、审核机制、竞拍规则、保证金说明 | Agent 1、2 | 3 天 |
| Agent 11 | 测试与验收 | 测试用例、接口测试、权限测试、竞价流程测试、端到端验收 | 全模块 | 5 天 |
| Agent 12 | 集成与发布 | 代码合并、接口对齐、环境配置、部署脚本、联调问题收口 | 全模块 | 5 天 |

## 4. 并行开发阶段安排

| 阶段 | 时间 | 并行任务 | 产出 |
|---|---|---|---|
| 第 1 阶段 | 第 1 至 2 天 | Agent 1、Agent 2 | 项目骨架、基础表结构、状态枚举、接口规范 |
| 第 2 阶段 | 第 3 至 7 天 | Agent 3、4、5、10 | 前台门户、拍品后台、企业认证、内容管理 |
| 第 3 阶段 | 第 6 至 10 天 | Agent 6、7、8、9 | 意向金审核、竞价核心、成交合同退款、个人中心消息 |
| 第 4 阶段 | 第 8 至 13 天 | Agent 11、12 | 测试用例、接口联调、权限验证、集成发布 |
| 第 5 阶段 | 第 14 至 15 天 | 全体 Agent 配合 | 缺陷修复、回归测试、验收材料整理 |

在多 Agent 并行顺利、需求不频繁变动的情况下，开发可按约 15 个工作日完成首轮可验收版本。若按自然月推进，剩余时间用于客户反馈、缺陷修复和验收调整。

## 5. Agent 任务细分

### 5.1 Agent 1：项目骨架与基础架构

工作内容：

- 初始化前端项目结构、后端项目结构和基础配置。
- 建立登录态、路由守卫、管理员与企业用户入口。
- 建立统一接口返回格式、错误码和分页格式。
- 建立文件上传基础能力，用于拍品图片、检测报告、营业执照、付款凭证。
- 建立操作日志基础中间件。

交付物：

- 可启动的前端与后端工程。
- 基础登录、权限校验、文件上传接口。
- 开发环境配置说明。

验收标准：

- 本地可启动。
- 前端可访问首页、登录页和后台入口。
- 后端健康检查、登录、文件上传接口可调用。

### 5.2 Agent 2：数据模型与状态机

工作内容：

- 设计拍品表、企业认证表、意向金表、出价记录表、成交结果表。
- 设计合同状态、退款状态、黑名单、通知、内容资讯、操作日志表。
- 定义拍品状态、认证状态、意向金状态、合同状态、退款状态枚举。
- 输出基础数据字典和字段说明。

交付物：

- 数据库迁移脚本。
- 模型定义。
- 状态枚举和字典接口。

验收标准：

- 数据表能正常创建。
- 状态枚举与 PRD 状态规则一致。
- 其他 Agent 可基于模型进行开发。

### 5.3 Agent 3：前台门户

工作内容：

- 首页门户：导航、搜索入口、登录/入驻入口、数据看板。
- 即将拍卖公告列表与详情。
- 正在竞价标的列表。
- 成交公示列表与详情。
- 矿产资源信息展示。
- 信息资讯入口和公开说明入口。

交付物：

- 前台页面与接口对接。
- 列表分页、关键词搜索、状态展示。
- 空状态和加载状态。

验收标准：

- 公众访客可查看公开信息。
- 首页数据看板可展示统计数据。
- 拍品卡片可进入详情页。

### 5.4 Agent 4：拍品与发布复核后台

工作内容：

- 拍品新建、编辑、保存草稿。
- 图片、附件、检测报告上传。
- 竞价规则、保证金规则、时间规则配置。
- 提交发布复核、审核通过、审核驳回。
- 发布后进入公示状态。
- 关闭/取消拍品。

交付物：

- 拍品管理后台页面。
- 拍品管理 API。
- 发布复核 API 与状态流转。

验收标准：

- 管理员可创建拍品并提交复核。
- 复核通过后前台可看到公示拍品。
- 审核驳回可记录驳回原因。

### 5.5 Agent 5：企业入驻与认证

工作内容：

- 企业注册与登录。
- 企业认证资料填写。
- 企业资质、营业执照等附件上传。
- 认证提交、待审核、审核通过、审核驳回。
- 驳回原因展示与重新提交。

交付物：

- 企业入驻页面。
- 企业认证后台审核页面。
- 认证状态接口。

验收标准：

- 企业可提交认证资料。
- 管理员可审核通过或驳回。
- 企业可查看认证状态。

### 5.6 Agent 6：意向金凭证与竞价资格

工作内容：

- 拍品详情页展示保证金金额和缴纳说明。
- 企业上传付款凭证。
- 管理员查看凭证、企业信息和应缴金额。
- 意向金审核通过、审核驳回。
- 审核通过后生成竞价资格。

交付物：

- 付款凭证上传页面。
- 意向金审核后台。
- 竞价资格校验接口。

验收标准：

- 认证通过企业可上传意向金凭证。
- 管理员审核通过后企业获得该拍品竞价资格。
- 未获资格企业不能报价。

### 5.7 Agent 7：竞价核心引擎

工作内容：

- 竞价详情页展示当前最高价、倒计时、加价幅度、保证金金额。
- 报价接口校验竞价状态、企业资格、黑名单、报价金额和加价规则。
- 记录出价序号、企业、金额、加价次数、出价时间、是否最高价。
- 当前最高价及时刷新。
- 竞拍截止后自动确认最高价企业中标。

交付物：

- 竞价详情页面。
- 报价 API。
- 出价记录 API。
- 中标确认任务。

验收标准：

- 有资格企业可成功报价。
- 低于规则报价被拒绝。
- 同时出价按服务器接收时间排序。
- 竞拍结束后生成成交结果。

### 5.8 Agent 8：成交、合同、退款与黑名单

工作内容：

- 生成成交公示。
- 展示成交拍品、中标企业名称、最终成交价。
- 后台登记合同状态：待签约、已签约、已完成、违约。
- 后台登记退款状态：未退款、审核中、已退款。
- 违约后关闭拍品、前台隐藏、企业可加入黑名单。
- 黑名单手动拉黑与解除。

交付物：

- 成交管理页面。
- 合同状态管理页面。
- 退款状态管理页面。
- 黑名单管理页面。

验收标准：

- 成交结果可公开展示。
- 合同状态为已完成后计入数据看板。
- 被拉黑企业不能参与竞拍。

### 5.9 Agent 9：个人中心与消息通知

工作内容：

- 企业个人中心。
- 查看认证状态、意向金状态、竞价记录。
- 查看站内消息。
- 竞拍结束后生成成交通知和失败通知。
- 短信通知接口预留发送记录。

交付物：

- 个人中心页面。
- 站内消息页面。
- 通知生成接口。

验收标准：

- 企业只能查看本企业数据。
- 中标企业收到成交通知。
- 未中标企业收到失败通知。

### 5.10 Agent 10：内容管理与公开说明页

工作内容：

- 政策法规、交易公告、矿能动态管理。
- 用户黑名单管理说明页。
- 信息发布审核机制说明页。
- 竞拍规则说明页。
- 保证金缴纳与退还说明页。

交付物：

- 内容管理后台。
- 资讯列表与详情页。
- 公开说明页。

验收标准：

- 管理员可维护资讯和说明内容。
- 前台可按分类查看资讯。
- 公开说明页可正常访问。

### 5.11 Agent 11：测试与验收

工作内容：

- 编写核心业务流程测试用例。
- 覆盖公众访客、企业用户、平台管理员三类角色。
- 覆盖拍品发布、企业认证、意向金审核、竞价、成交、合同、退款、黑名单。
- 执行权限测试和异常测试。

交付物：

- 测试用例清单。
- 缺陷记录。
- 回归测试结果。

验收标准：

- 主流程可完整跑通。
- 权限边界符合 PRD。
- 关键异常场景有明确提示。

### 5.12 Agent 12：集成与发布

工作内容：

- 合并各 Agent 代码。
- 对齐接口字段和状态枚举。
- 处理跨模块冲突。
- 配置测试环境和生产环境。
- 整理部署说明和验收说明。

交付物：

- 可部署版本。
- 环境配置文档。
- 联调问题清单。
- 验收交付包。

验收标准：

- 系统可在目标环境启动。
- 前后台主流程联调通过。
- 验收材料完整。

## 6. 推荐并行分组

为减少冲突，建议按以下分组并行：

| 分组 | Agent | 说明 |
|---|---|---|
| 基础组 | Agent 1、Agent 2 | 先行完成基础设施与数据模型 |
| 前台组 | Agent 3、Agent 9、Agent 10 | 面向公众和企业用户页面 |
| 后台组 | Agent 4、Agent 5、Agent 6、Agent 8 | 面向管理员操作与审核 |
| 交易核心组 | Agent 7 | 专注竞价逻辑，避免被其他模块干扰 |
| 质量与集成组 | Agent 11、Agent 12 | 持续测试、合并、部署和收口 |

## 7. 关键依赖关系

| 前置模块 | 依赖模块 | 原因 |
|---|---|---|
| 项目骨架与基础架构 | 全部模块 | 统一工程结构、权限、上传和接口格式 |
| 数据模型与状态机 | 全部业务模块 | 统一字段、状态和数据关系 |
| 拍品与发布复核 | 前台门户、意向金、竞价核心 | 拍品是公告、报名和竞价基础 |
| 企业入驻与认证 | 意向金、竞价核心、个人中心 | 企业认证决定后续操作权限 |
| 意向金凭证与资格 | 竞价核心 | 资格审核决定是否可报价 |
| 竞价核心 | 成交、合同、退款、通知 | 竞价结果驱动成交和消息 |

## 8. 开发时间预估

在多 Agent 并行协作下：

| 工作项 | 时间 |
|---|---|
| 基础架构与数据模型 | 2 个工作日 |
| 业务模块并行开发 | 8 个工作日 |
| 集成联调与测试 | 4 个工作日 |
| 缺陷修复与验收收口 | 3 个工作日 |
| 合计 | 约 17 个工作日 |

如按自然月安排，建议预留 1 个月用于开发、联调、客户反馈和验收调整。

## 9. 交付顺序建议

1. 先交付项目骨架、数据模型和登录权限。
2. 再交付拍品管理、企业认证、意向金审核。
3. 再交付竞价核心、成交公示、合同退款和黑名单。
4. 再交付前台门户、个人中心、内容管理和数据看板。
5. 最后进行全流程联调、权限测试和验收修复。

## 10. 总结

该拍卖交易系统适合拆为 12 个 Agent 并行开发。核心原则是先统一数据模型和状态规则，再让前台、后台、竞价、内容、测试、集成分别推进。按多 Agent 协作方式，首轮可验收版本预计约 17 个工作日完成，按项目管理口径建议按 1 个月安排整体交付。
