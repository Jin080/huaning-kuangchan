# 华宁矿产竞拍平台 PC 端 PRD

## 1. 项目概述

### 1.1 项目背景

建设一套参考“贵阳市矿产品数字化交易平台”门户结构与政务交易风格的 PC 端矿产竞拍平台。平台由华宁国企或平台运营方统一发布矿产拍品信息，企业用户完成商家入驻认证、缴纳线下意向金并通过审核后，在线参与竞价。系统负责公告展示、资格审核、线上竞价、成交公示、通知、状态留痕和后台管理；合同签署、尾款支付、保证金退款均在线下完成。

### 1.2 建设目标

- 建立公平、公正、公开的矿产竞拍信息平台。
- 支撑平台管理员发布拍品、公示拍品、审核企业与意向金。
- 支撑企业用户查看公告、完成认证、上传意向金凭证、参与竞价。
- 支撑竞拍结束后自动确认最高价中标，并生成成交公示。
- 支撑合同状态、退款状态、黑名单、数据看板等后台管理能力。

### 1.3 当期开发内容

当期开发内容包括：

- 首页门户
- 数据看板
- 矿产资源信息展示
- 即将拍卖公告
- 正在竞价标的
- 竞价详情
- 成交公示
- 信息资讯
- 公开说明页
- 企业入驻/认证
- 个人中心
- 管理后台

## 2. 用户角色

### 2.1 公众访客

- 可查看首页、公开公告、矿产资源信息、竞价列表、成交公示、信息资讯、公开说明页。
- 不可报名、上传凭证、参与竞价。

### 2.2 企业用户

- 可注册/入驻、提交企业认证材料。
- 认证通过后可提交意向金付款凭证。
- 意向金审核通过后可参与对应拍品竞价。
- 可查看本企业认证状态、意向金状态、竞价记录、站内通知。

### 2.3 平台管理员

- 管理拍品、公告、资讯、公开说明页。
- 进行企业认证复核、意向金复核、标的发布复核。
- 管理竞拍结果、合同状态、退款状态、黑名单。
- 查看后台数据统计和操作留痕。

## 3. 核心业务流程

### 3.1 标的发布流程

1. 管理员创建拍品草稿。
2. 管理员填写拍品信息、竞价规则、客户须知、保证金规则、时间规则。
3. 管理员提交标的发布复核。
4. 复核人员审核。
5. 审核通过后，拍品进入公示期，并展示在“即将拍卖公告”。
6. 审核驳回时，填写驳回原因，拍品退回草稿/待修改状态。

发布前需弹出确认框：

> 发布后该拍品将进入公示期，企业可查看公告并提交意向金凭证。请确认竞拍时间、起拍价、加价幅度、保证金比例、客户须知、延时规则等信息已填写完整。

### 3.2 企业认证流程

1. 企业用户进入商家入驻页面。
2. 填写企业认证信息并上传附件。
3. 提交后进入“待审核”。
4. 管理员人工审核。
5. 审核通过后，企业获得提交意向金凭证资格。
6. 审核驳回时，展示驳回原因，企业可修改后重新提交。

### 3.3 意向金审核流程

1. 企业用户在公示期查看拍品详情。
2. 用户阅读客户须知。
3. 用户按页面展示的线下付款信息缴纳意向金/保证金。
4. 用户上传付款凭证。
5. 管理员人工复核付款凭证。
6. 审核通过后，企业获得该拍品竞价资格。
7. 审核驳回时，展示驳回原因，企业可重新上传凭证。

### 3.4 竞价流程

1. 公示期结束后，拍品按配置时间进入竞拍期。
2. 拍品展示在“正在竞价标的”。
3. 仅企业认证通过且该拍品意向金审核通过的企业可报价。
4. 用户只能看到当前最高价，不显示最高价企业名称。
5. 用户按管理员配置的加价幅度和加价次数进行报价。
6. 系统以服务器收到报价的时间作为报价顺序。
7. 竞拍截止后，系统自动以最高价确认中标企业。
8. 系统生成成交结果，并发送成交通知和失败通知。

延时竞价为备选配置：后台可启用或不启用。若启用，必须在拍卖前对用户公开展示延时规则。

### 3.5 成交与合同流程

1. 竞拍结束后系统生成成交公示。
2. 成交公示公开中标企业名称、最终成交价、成交拍品。
3. 中标企业线下签合同、补齐尾款。
4. 管理员在后台登记合同状态。
5. 合同状态为“已完成”后，成交额计入数据看板。
6. 若中标企业违约，管理员标记违约，保证金不退，可拉黑账号。
7. 违约拍品后台关闭，前台不显示，不进行二次竞拍。

### 3.6 未中标退款流程

1. 竞拍结束后，未中标企业收到失败通知。
2. 通知说明保证金将退回。
3. 线下将保证金打回企业账户。
4. 后台记录退款状态。

退款状态：未退款、审核中、已退款。

系统不要求上传退款凭证。

## 4. 前台页面需求

### 4.1 首页

首页应参考贵阳市矿产品数字化交易平台的门户风格，展示交易入口、数据概览、矿产资源、公告资讯。

模块：

- 顶部导航
- 搜索入口
- 登录/入驻入口
- 数据看板
- 即将拍卖公告
- 正在竞价
- 成交公示
- 矿产资源信息展示
- 信息资讯
- 公开说明入口

数据看板指标：

- 本年成交量
- 本年成交额
- 累计成交量
- 累计成交额

数据统计口径：

- 本年成交量：合同状态为“已完成”且完成时间属于本年的成交拍品数量。
- 本年成交额：合同状态为“已完成”且完成时间属于本年的成交价总和。
- 累计成交量：所有合同状态为“已完成”的成交拍品数量。
- 累计成交额：所有合同状态为“已完成”的成交价总和。
- 已竞拍结束但合同未完成的拍品，不计入成交额。

## 9. 权限规则

- 公众访客只能查看公开信息。
- 企业用户必须完成认证后，才能提交意向金凭证。
- 企业用户必须完成对应拍品意向金审核后，才能参与该拍品竞价。
- 企业用户只能查看本企业认证、报名、意向金、出价、通知等数据。
- 管理员可查看全量数据。
- 被拉黑企业账号封禁，不允许继续参与竞拍。

## 10. 异常处理

- 未登录用户点击报名或报价，引导登录。
- 未认证企业点击报名，提示先完成企业认证。
- 认证审核中企业点击报名，提示认证审核中。
- 意向金未审核通过企业点击报价，提示暂无竞价资格。
- 被拉黑企业登录或操作，提示联系平台客服电话。
- 竞拍已结束后提交报价，提示竞拍已结束。
- 报价低于当前最高价或不符合加价规则，提示重新报价。
- 同时出价时，以服务器接收时间为准，后到请求需基于最新最高价重新报价。
- 拍品违约或取消后，前台不再展示。

## 11. 非功能需求

### 11.1 安全

- 关键操作需记录操作日志。
- 出价记录不可由普通管理员删除。
- 企业认证资料、付款凭证等敏感附件需做权限控制。
- 后台审核、拉黑、合同状态变更需记录操作人和操作时间。

### 11.2 性能

- 首页优先加载导航、数据看板、公告、竞价入口。
- 列表页支持分页。
- 竞价页当前最高价需及时刷新。

### 11.3 兼容

- PC 端适配主流 Chrome、Edge 浏览器。
- PC 端适配主流 Chrome、Edge 浏览器。
